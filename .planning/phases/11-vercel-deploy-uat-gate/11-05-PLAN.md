<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Deployed `*.vercel.app` URL | Lighthouse audits from local machine over public internet; throttling settings simulate mobile network |
| Lighthouse report JSON/HTML | Contains the audited URL + page structure + perf timings — already public info |
| HARD gate verdict | Single source of truth for Phase 12 unblock; tampering would let a failing build proceed to canonical domain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05-01 | T (Tampering) | Lighthouse JSON report between `lighthouse:deployed` and `lighthouse:check` | accept | Same-machine sequential execution. Local trust boundary. If local malware can modify the JSON between run and check, larger compromise already in play. |
| T-11-05-02 | R (Repudiation) | Gate verdict accuracy | mitigate | Verdict log embeds JSON-derived scores (Task 3 uses node to extract from `lighthouse-report-mobile.report.json`), and the HTML report is committed alongside. Reviewer can re-run Lighthouse against the same deployed URL to corroborate. Git commit hash + timestamp anchor the audit. |
| T-11-05-03 | D (DoS) | Audit fails due to flaky network / Vercel edge cache cold-start | mitigate | If Lighthouse fails for non-quality reasons (network timeout, Vercel cold-start cache), Task 1 instructions cover re-running. Multiple consecutive runs at different times-of-day reduce false-fail noise. (Best practice for first audit: warm the edge with a `curl` before Lighthouse — included in Task 1 sanity-check.) |
| T-11-05-04 | I (Info Disclosure) | Lighthouse report HTML contains page screenshots | accept | Site is public-facing portfolio. No private content to leak. |
| T-11-05-05 | S (Spoofing) | Wrong URL audited (audit hit localhost instead of deployed) | mitigate | Task 1 `<verify>` automated check parses report JSON and asserts `finalUrl.includes('.vercel.app')`. Cannot accidentally commit a localhost audit as a deployed-URL gate result. |

All LOW severity. Highest concern (T-11-05-05) is mitigated by hard automated check.
</threat_model>
