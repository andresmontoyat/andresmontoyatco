# Phase 16 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope of This Contract

Phase 16 ships two new surfaces that plug into Phase 15's locked renderer contract:

1. **SkillFilters bar** — multi-skill chip cluster, year-range dual-thumb slider, 8 category chips, reset button. Desktop: persistent bottom bar. Mobile: compact top toolbar with inline year mini-control.
2. **ExperienceCard** — `role="dialog"` floating panel; single scrollable job list; tech chips (click-to-filter); CV CTA; focus trap; Esc close; mobile bottom-sheet variant.

Phase 15's design system, tokens, keyframes, and focus-ring pattern are inherited without modification. This contract adds only the net-new tokens and keyframes required for chips, slider, card surfaces, and entry/exit animations.

## Animation Contract

### New keyframes to add to `tailwind.config.js`

```js
// Add to theme.extend.keyframes:
cardFadeIn: {
  '0%':   { opacity: '0', transform: 'scale(0.96)' },
  '100%': { opacity: '1', transform: 'scale(1.0)' },
},
cardSlideUp: {
  '0%':   { opacity: '0', transform: 'translateY(100%)' },
  '100%': { opacity: '1', transform: 'translateY(0%)' },
},
cardSwapOut: {
  '0%':   { opacity: '1' },
  '100%': { opacity: '0' },
},
cardSwapIn: {
  '0%':   { opacity: '0' },
  '100%': { opacity: '1' },
},
chipFlash: {
  '0%':   { opacity: '1' },
  '50%':  { opacity: '0.5', transform: 'scale(0.94)' },
  '100%': { opacity: '1', transform: 'scale(1.0)' },
},

// Add to theme.extend.animation:
'card-fade-in':  'cardFadeIn 200ms cubic-bezier(0.22, 1, 0.36, 1) both',
'card-slide-up': 'cardSlideUp 250ms cubic-bezier(0.22, 1, 0.36, 1) both',
'card-swap-out': 'cardSwapOut 75ms ease-in both',
'card-swap-in':  'cardSwapIn 75ms ease-out both',
'chip-flash':    'chipFlash 100ms ease-out both',
```

**Easing rationale:**
- Card open (desktop + mobile): `cubic-bezier(0.22, 1, 0.36, 1)` — "ease-out expo" variant. Fastest initial movement, settles crisply. Signals responsiveness without bounce.
- Card swap: 75ms out + 75ms in = 150ms total cross-fade. Two separate `opacity`-only keyframes. Purely GPU-composited — no layout thrash. Content swaps at the 75ms midpoint.
- Chip flash: 100ms `ease-out` scale + opacity pulse. Sub-threshold for perceived latency.

### Full animation table

| Event | Duration | Easing | Keyframe | Class |
|-------|----------|--------|----------|-------|
| Card open — desktop | 200ms | `cubic-bezier(0.22, 1, 0.36, 1)` | `cardFadeIn` | `motion-safe:animate-card-fade-in` |
| Card open — mobile bottom-sheet | 250ms | `cubic-bezier(0.22, 1, 0.36, 1)` | `cardSlideUp` | `motion-safe:animate-card-slide-up` |
| Card swap out (current content) | 75ms | `ease-in` | `cardSwapOut` | `motion-safe:animate-card-swap-out` |
| Card swap in (new content) | 75ms | `ease-out` | `cardSwapIn` | `motion-safe:animate-card-swap-in` |
| Tech chip click-to-filter flash | 100ms | `ease-out` | `chipFlash` | `motion-safe:animate-chip-flash` |
| Theme/lang switch on filter bar | 200ms | `ease` | CSS transition | `motion-safe:transition-colors duration-200` (existing global) |
| Mobile overlay fade in | 200ms | `ease-out` | CSS transition | `motion-safe:transition-opacity duration-200` |
| Reset button enable/disable | 150ms | `ease` | CSS transition | `motion-safe:transition-opacity duration-150` |

### Reduced-motion path

The global `src/index.css` `@media (prefers-reduced-motion: reduce)` rule suppresses all `animation-duration` to `0.01ms`. Phase 16 additionally:

1. **Card open:** No fade/slide animation. Card renders immediately at full opacity (`opacity: 1`) and final position. Never show a half-rendered card.
2. **Card swap:** No cross-fade. Content updates synchronously on the same frame. Focus migrates immediately to new card heading.
3. **Chip flash:** No scale/opacity animation. The chip simply updates its visual state (active/outline) instantly.
4. **Mobile bottom-sheet:** No `translateY` slide. Bottom-sheet renders at final position immediately.
5. **Dim treatment on filter change:** `fill-opacity` on nodes updates instantly (no CSS transition on `fill-opacity`). Under motion-safe, a `200ms ease-out` transition is applied via `style={{ transition: 'fill-opacity 200ms ease-out' }}` on each `<circle>` element; under `prefersReducedMotion`, `transition: 'none'`.

All animation classes MUST carry the `motion-safe:` prefix. Example: `motion-safe:animate-card-fade-in`. Never use bare animation class names on Phase 16 components.

### Card swap implementation contract

```
User clicks node B while card (node A) is open:
  1. Add motion-safe:animate-card-swap-out to card content wrapper → 75ms fade-out
  2. onAnimationEnd: update card content to node B's job data
  3. Add motion-safe:animate-card-swap-in to card content wrapper → 75ms fade-in
  4. Total: 150ms perceived swap

Under reduced-motion:
  1. Update card content to node B's data synchronously
  2. Move focus to new card heading immediately
  No animation steps.
```

| `ArrowLeft` / `ArrowDown` | Decrement active thumb −1 year |
| `Home` | Jump active thumb to its range minimum |
| `End` | Jump active thumb to its range maximum |
| `Tab` | Move focus to next element (next thumb, or next focusable outside slider group) |

`e.preventDefault()` on all arrow key and Home/End events to prevent page scrolling.

Start thumb: `aria-valuemin = YEAR_MIN`, `aria-valuemax = endYear − 1` (cannot exceed end thumb).
End thumb: `aria-valuemin = startYear + 1`, `aria-valuemax = YEAR_MAX` (cannot go below start thumb).
Both thumbs: announce updated `aria-valuenow` + `aria-valuetext` on every keydown/drag update.

**Reset button `aria-disabled` vs `disabled` note:** Use both `disabled` (native) and `aria-disabled="true"` on the reset button. Native `disabled` removes it from tab order, but since it is the last element in the filter bar, removing it from tab order is acceptable (keyboard users will Tab past it when no filter is active). If the UX decision changes to keep reset always tabbable, replace `disabled` with `aria-disabled` only and handle click suppression in JS.

### Focus trap contract

Focus trap is implemented inline (no external library), mirroring the `MobileMenu` pattern in `Nav.js`.

```
Tab cycle (forward):
  h2 (skill heading, tabIndex=-1, initial focus) →
  [tech chips inside job entries, in DOM order] →
  close button (X) →
  CV CTA download link →
  back to first focusable (first tech chip OR close button if no tech chips)

Shift+Tab cycle (backward):
  Reverse of above.
```

**Focus trap implementation:**
```js
function handleCardKeyDown(e) {
  if (e.key === 'Escape') {
    closeCard()           // closes card + clears selection
    returnFocusToOrigin() // calls rovingEntryRef.current?.focus() from Phase 15
    return
  }
  if (e.key === 'Tab') {
    const focusable = cardRef.current.querySelectorAll(
      'button:not([disabled]), a[href], [tabindex="-1"][data-focus-trap-start]'
    )
    // The heading has data-focus-trap-start to be included only in the cycle
    const first = focusable[0]
    const last = focusable[focusable.length - 1]
    if (e.shiftKey && document.activeElement === first) {
      e.preventDefault()
      last.focus()
    } else if (!e.shiftKey && document.activeElement === last) {
      e.preventDefault()
      first.focus()
    }
  }
}
```

**Overlay click close:** a full-screen `div` behind the card surface (not inside the dialog) carries `onClick={closeCard}` and `aria-hidden="true"`. It must NOT be a sibling inside `role="dialog"` — it sits outside the dialog element to avoid confusing SR users.

**Return focus:** on close, call `rovingEntryRef.current?.focus()` — Phase 15 already manages this ref on the `role="application"` div. Phase 16 receives `rovingEntryRef` as a prop from `GameMode` and calls its `.focus()` on dialog close.

**`document.body` overflow lock on mobile sheet open:** set `document.body.style.overflow = 'hidden'` on open (mirrors `MobileMenu`); restore on close.

## Empty-State Contract

When the combined filter state (selectedSkills ∩ yearRange ∩ category) yields zero matching skills or zero matching job entries:

### Zero matching skills (all nodes dim)

1. All constellation nodes dim to `fill-opacity: 0.35` (same post-selection dim treatment from Phase 15).
2. An inline message renders between the H1 block and the constellation: centered `<p>` with `t.game.filterEmpty` + `t.game.filterEmptyHint`.
3. The Reset button gains `className="... border border-brand text-brand"` (brand-colored outline instead of default outline) to signal it as the recommended action. The border width increases to 2px on empty state.
4. `aria-live="polite"` region announces: `t.game.filterEmpty` (same region used by Phase 15's skill selection announcements — reuse).
5. ExperienceCard does NOT open (no skill selected in the visual sense when all are dimmed). If a card was open when filters made all nodes zero-match, close the card.

### Zero matching jobs (skill selected but no jobs pass year/category filter)

When a skill IS selected (card is open) but the intersection of that skill's jobs with the active filters returns empty:

1. ExperienceCard remains open.
2. The `<ol>` job list is replaced with: `<p className="text-text-secondary text-sm px-4 py-8 text-center">{t.game.filterEmpty}</p>`.
3. CV CTA remains visible in the footer (user might still want to download the CV).
4. The `aria-label` on the job list becomes `t.game.filterEmpty` in this state.

## Tech-Chip Click-to-Filter Contract

Tech chips inside ExperienceCard job entries function as secondary filter toggles.

| State | Visual | ARIA |
|-------|--------|------|
| Inactive (not in selectedSkills) | Outline chip: `border-chip-outlineBorder text-chip-outlineText bg-transparent` | `aria-pressed="false"` |
| Active (in selectedSkills) | Filled chip: `bg-chip-activeBg text-chip-activeText` | `aria-pressed="true"` |
| Active and locked (the skill whose card is currently open) | Filled chip + cursor-default + `opacity-75` | `aria-pressed="true"`, `aria-disabled="true"`, `aria-describedby` pointing to sr-only text `t.game.skillActiveAndLocked` |

**Toggle behavior:**
- Click on inactive chip → adds skill to `selectedSkills` → fires `motion-safe:animate-chip-flash` → updates constellation dim state.
- Click on active chip (not locked) → removes skill from `selectedSkills` → updates dim state.
- Click on locked chip → no-op (the currently selected skill cannot be deselected from within the card; user must close card or click empty space per D-16-FLOW-CLOSE-EMPTY).

**Chip click constellation feedback:** adding a skill via a tech chip causes the newly highlighted node to briefly gain the halo treatment (same `filter: drop-shadow(...)` as an active node). Under reduced-motion, only `fill-opacity` updates — no filter effect.

## Card Swap Contract

Clicking a different skill node while ExperienceCard is open:

```
1. New node click fires onSelectSkill(newNodeId) via SvgConstellation.
2. useConstellation toggles selectedSkillId to newNodeId.
3. ExperienceCard receives new selectedSkillId prop.
4. Motion-safe path:
   a. Apply motion-safe:animate-card-swap-out to card content wrapper (75ms, opacity 0→1 reversed → 1→0).
   b. onAnimationEnd: set card content state to new node's data.
   c. Apply motion-safe:animate-card-swap-in (75ms, opacity 0→1).
   d. Move focus to new card h2 heading via headingRef.current.focus().
5. Reduced-motion path:
   a. Update card content state to new node's data immediately.
   b. Move focus to new card heading.
6. Constellation: halo moves to new node, previous node dims to 0.35 (or 1.0 if still in selectedSkills).
```

The card position on desktop recalculates to be near the new node. Under reduced-motion, position updates instantly. Under motion-safe, the card fades out at old position and fades in at new position (same `cardFadeIn` keyframe reused for reposition).
