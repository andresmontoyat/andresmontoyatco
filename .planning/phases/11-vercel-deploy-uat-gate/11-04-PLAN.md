    curl -sI "$VERCEL_URL/some-random-deep-path" | grep -iE '^(HTTP/|content-type)'
    # Expect: HTTP/2 200, content-type: text/html (rewrite to /index.html worked)

    curl -sI "$VERCEL_URL/" | head -3

    # First find a hashed bundle name from the deployed site:

    echo "Sampled bundle: $BUNDLE"
    curl -sI "$VERCEL_URL$BUNDLE" | grep -iE '^(HTTP/|cache-control)'
    # Expect: HTTP/2 200, Cache-Control: public, max-age=31536000, immutable

    # 5. No 404 for hash-anchor scenario in browser (manual confirmation captured in Task 5)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vercel CLI auth tokens | Stored in `~/.local/share/com.vercel.cli/` (NOT in repo); leak = full account compromise |
| `.vercel/project.json` | Contains projectId + orgId; gitignored per 11-01 to prevent commit |
| Vercel build pipeline | Runs `npm install` + `npm run build`; npm scripts could exfiltrate (low risk — this is our own code) |
| Public `*.vercel.app` URL | Anyone can reach; assumed-public (this is the point — recruiter-facing portfolio) |
| Vercel Dashboard (Settings → Git) | Misconfig (wrong production branch, ignored build step set) breaks auto-deploy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04-01 | I (Info Disclosure) | Vercel CLI auth token leak | mitigate | Token lives in OS-keychain-equivalent (`~/.local/share/com.vercel.cli/`). 11-01 Task 2 added `.vercel/` to `.gitignore` to prevent the local project link file (projectId+orgId) from being committed. Task 2 here explicitly verifies `git check-ignore -v .vercel/`. |
| T-11-04-02 | T (Tampering) | Wrong production branch / ignored build step in Vercel dashboard | mitigate | Task 5 Part A explicitly verifies "Production Branch: main" and "Ignored Build Step: default" in dashboard. Future pushes to `main` will auto-deploy; pushes to feature branches generate previews (Phase 13 territory). |
| T-11-04-03 | E (Elevation) | Build pipeline runs arbitrary npm scripts | accept | We control the package.json scripts; risk is from supply-chain attack on npm deps, NOT from our pipeline. Out-of-scope for Phase 11. |
| T-11-04-04 | I (Info Disclosure) | `og:url` still points to `andresmontoyat.co` but live URL is `*.vercel.app` (D-12) | accept | D-12 explicitly accepts this. Share-debugger on Vercel URL will flag mismatch — acceptable for Phase 11 (deploy validation). Phase 12 lands the canonical domain. Mitigation deferred to Phase 12, not this phase. |
| T-11-04-05 | D (DoS) | Vercel free-tier deploy quotas / bandwidth | accept | Personal portfolio; expected traffic is low (single-digit visits/day typically). Quotas are not a realistic concern. |
| T-11-04-06 | S (Spoofing) | `*.vercel.app` URL hijacking via stolen Vercel token | mitigate | Same as T-11-04-01: keep `.vercel/` gitignored; rely on Vercel's account-level 2FA (out-of-scope for this plan, but recommended user-side). |

LOW severity overall. Highest concern (T-11-04-01) is mitigated by 11-01 Task 2 + verification in Task 2 here.
</threat_model>
