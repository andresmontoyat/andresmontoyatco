<!-- Font preload contract — three woff2 files, all latin subset -->

The @fontsource package ships per-weight CSS files which @import the woff2 files
from the same package's `files/` subdirectory. Vite resolves these at build time
and emits them to `dist/assets/` with content-hashed filenames (e.g.
`inter-latin-400-normal-abc123.woff2`).

For preload, the source path that index.html references must match the path
Vite emits in dev mode AND production. Vite resolves `/node_modules/@fontsource/inter/files/inter-latin-400-normal.woff2` to a hashed asset URL at build time only when the path is referenced from JS or CSS — `<link>` tags in index.html are NOT processed by Vite's asset pipeline by default.

THE CORRECT PATTERN FOR @fontsource PRELOAD IN VITE: Reference the woff2 file via Vite's `?url` import suffix or use a build-time constant. BUT for this codebase's simplicity, an alternative is to copy the 3 woff2 files into `public/fonts/` (Vite serves public/ as-is at root) and preload from `/fonts/inter-latin-400-normal.woff2`. The downside: the @fontsource CSS still requests its own copy from the hashed dist asset, so the preload would not match the actual fetch URL — making it useless or even harmful (browser warning "preload was not used").

Resolution chosen for this plan: Use Vite's recommended pattern — `import` the woff2 URL at the JS layer (so Vite hashes it) and inject the preload link via vanilla DOM in src/index.js BEFORE React mounts. This guarantees the preload URL matches the actual fetched URL.

```js
// In src/index.js, BEFORE the React DOM mount:
import inter400Url from '@fontsource/inter/files/inter-latin-400-normal.woff2?url'
import inter800Url from '@fontsource/inter/files/inter-latin-800-normal.woff2?url'
import jbm400Url from '@fontsource/jetbrains-mono/files/jetbrains-mono-latin-400-normal.woff2?url'

function preloadFont(href) {
  const link = document.createElement('link')
  link.rel = 'preload'
  link.as = 'font'
  link.type = 'font/woff2'
  link.crossOrigin = 'anonymous'
  link.href = href
  document.head.appendChild(link)
}

[inter400Url, inter800Url, jbm400Url].forEach(preloadFont)
```

This guarantees the preload URL == the @fontsource CSS fetch URL because both reference the same Vite-hashed asset.

ALTERNATIVE simpler pattern (chosen for this plan because it's static-html-friendly and keeps src/index.js minimal): add three `<link>` tags directly in index.html that point to root-relative paths Vite preserves through the build. This requires verifying Vite's behavior — see Task 1 step-by-step.
