## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local production build (`dist/`) → tester's browser | Same boundary as Vercel will use; verifying here catches regressions before public deploy |
| Tester observations → audit log | Honesty / completeness of recorded results; lying or skipping = letting a regression onto Vercel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03-01 | R (Repudiation) | UAT pass/fail verdict | mitigate | Audit log `.planning/phases/11-vercel-deploy-uat-gate/11-UAT-LOCAL.md` is committed to git after Task 9 — provides audit trail of who tested what, when, against which build. |
| T-11-03-02 | T (Tampering) | dist/ build output between build + serve | accept | Local-only; same trust boundary as developer's working copy. If local malware modified `dist/` we have bigger problems than UAT. |
| T-11-03-03 | I (Info Disclosure) | DevTools screenshots in audit log | accept | Tester may attach screenshots; local-only environment; no production secrets visible (theme colors, hero photo are public artifacts). |
| T-11-03-04 | D (DoS) | UAT skipped under time pressure | mitigate | Audit log explicitly requires `passed + failed + skipped + pending` accounting. `Closure` section forces a verdict (ALL PASS or BLOCKED). 11-04 plan requires `all-pass` resume signal — orchestrator cannot proceed silently. |

All LOW severity. Phase 11-03 is human-driven local verification — primary "threat" is testing discipline, mitigated by audit-log structure.
