# Phase 21 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

Phase 21 is an infrastructure/routing phase (Astro scaffold, `astro:i18n`, Vercel Edge Middleware, `BaseLayout.astro`, deploy cleanup). It has exactly **two** visible UI surfaces:

1. **`404.astro`** (new page) — the only genuinely new visual/copy decision this phase makes. Locked by CONTEXT.md **D-10** (single generic route, no per-locale duplication) and **D-11** (minimal design, existing dark-palette tokens only, bilingual message, link back to `/`, no new animation/personality).
2. **`BaseLayout.astro`** (new file, no new visual design) — a structural port of the existing `index.html` `<head>` (favicon, viewport, theme-color, OG/Twitter, JSON-LD, critical hero CSS, GA script; CONTEXT.md **D-05..D-09** already lock all of this) plus one new blocking inline theme-flip `<script is:inline>` (**ISLAND-04**). No new colors, spacing, or typography are introduced by `BaseLayout.astro` — it renders no visible body content of its own (`<slot />` only).

Everything else in this phase (`astro.config.mjs`, `middleware.ts`, `vercel.json`, `package.json`) has zero visual surface.

**Governing rule for this UI-SPEC: reuse 100% of the existing Tailwind CSS-variable token system verbatim. Do not invent new visual language.** All values below are pulled directly from `tailwind.config.js` and `src/index.css` (dark palette shipped in v4.0, `:root` block), not proposed as new tokens.

## Typography

Declared for the 404 page only (`BaseLayout.astro` renders no visible text of its own). Exactly 4 sizes, exactly 2 weights — both pulled from the existing scale already used across `Hero.jsx`/`Nav.jsx`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 regular (`font-normal`) | 1.5 |
| Label (eyebrow, e.g. "404 · Not Found") | 12px (`text-xs`) | 800 extrabold (`font-extrabold`), `uppercase tracking-wide`, `font-mono` | 1.4 |
| Heading (bilingual "Page not found" / "Página no encontrada") | 24px (`text-2xl`) | 800 extrabold (`font-extrabold`) | 1.2 |
| Display (large "404" numeral) | 64px (`text-6xl`) | 800 extrabold (`font-extrabold`) | 1.1 |

Only two weights used site-wide already (400 / 800) — reused exactly, no new weight introduced. The `font-pixel` ("Press Start 2P") retro display font used elsewhere for hero titles is **deliberately not used here** — CONTEXT.md D-11 explicitly rules out "new animation/personality work" for this foundation-phase page.

## Color

All values are existing CSS custom properties from `src/index.css` `:root` (dark theme — the site's default `color-scheme: dark`). No new hex values introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` → `#0B1020` (Tailwind `bg-ink-900` / `bg-bg`) | Full-page background |
| Secondary (30%) | `var(--color-ink-800)` → `#0F1530` (Tailwind `bg-ink-800`) | Optional subtle content-block backdrop only if a wrapper is used (not required — content can sit directly on `bg-ink-900`) |
| Accent (10%) | `var(--bg-brand-gradient)` (`linear-gradient(135deg, #3B82F6 0%, #10B981 100%)`, Tailwind `bg-brand-gradient`) applied via `bg-clip-text text-transparent` | Reserved for exactly two elements: (1) the large "404" numeral display text, (2) the "back to home" CTA link's hover border/text color (`hover:border-brand hover:text-brand`, matching `Hero.jsx`'s existing secondary-link pattern) |
| Destructive | not applicable | No destructive actions exist in this phase (routing/scaffold phase; 404 page has one non-destructive link) |

Accent reserved for: the 404 numeral gradient text, and the "back to home" link's hover/focus state only. Not applied to body copy, the eyebrow label, or the page background.

## Copywriting Contract

Bilingual (EN primary rows, ES secondary) per project-wide i18n constraint. 404 stays locale-agnostic in its own route (D-10) but infers a message language from the broken URL's `/en`/`/es` prefix when present, defaulting to English otherwise.

| Element | Copy |
|---------|------|
| Primary CTA (EN) | "Back to home" — `<a href="/en">` if the broken path started with `/en`, else `<a href="/">` |
| Primary CTA (ES) | "Volver al inicio" — `<a href="/es">` if the broken path started with `/es` |
| Eyebrow label (EN) | "404 · Not Found" |
| Eyebrow label (ES) | "404 · No encontrado" |
| Heading (EN) | "Page not found" |
| Heading (ES) | "Página no encontrada" |
| Body / empty-state style copy (EN) | "The page you're looking for doesn't exist or has moved. Let's get you back on track." |
| Body / empty-state style copy (ES) | "La página que buscas no existe o fue movida. Volvamos al inicio." |
| Error state | This page IS the error state — no separate error-within-error copy needed. No form, no retry action; the sole recovery path is the "back to home" CTA above. |
| Destructive confirmation | Not applicable — no destructive actions in this phase. |

**Locale-inference note (implementation detail, not a design decision):** if `Astro.url.pathname` starts with `/es`, render the ES copy + `/es` CTA target; if it starts with `/en`, render the EN copy + `/en` CTA target; otherwise (bare unknown path, e.g. `/foo`) default to EN copy + `/` CTA target (root, which the Phase 21 middleware then redirects per ROUTE-02). This satisfies D-11's "locale-inferred from the broken URL's `/en`/`/es` prefix if present" without creating per-locale route files (D-10).
