Mi práctica con Claude Code

Llevo meses construyendo software con Claude Code como peer engineer — no como autocomplete. Mi enfoque combina hexagonal architecture, atomic commits, y verification gates en cada paso.

He construido un sistema de workflows (GSD — Get Shit Done) que disciplina el ciclo: discuss → plan → execute → verify, con artefactos en .planning/, ADRs en docs/adr/, y subagents especializados (build-validator, code-reviewer, tdd-guide, doc-classifier).

Resultados reales:

  • 39+ subagents custom — desde clasificadores de docs hasta milestone auditors
  • 50+ skills integradas, incluyendo caveman (compresión ~75% tokens) y spotless
  • MCP servers conectando Spring AI a Qdrant (RAG production-ready)
  • Apps end-to-end entregadas con loops agénticos — claude-kanban, este mismo portfolio

El common thread: AI no reemplaza ingeniería — la amplifica cuando hay disciplina arquitectónica alrededor.
