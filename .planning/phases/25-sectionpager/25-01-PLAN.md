<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Astro server render → hydrated island props | `locale` is set at build time from `Astro.currentLocale` (allowlisted `/en`/`/es` only by astro:i18n config), serialized into the island. Not runtime user input. |
| Browser DOM ↔ island | scroll/resize listeners read `window.scrollY`/`scrollHeight`; `scrollToId` reads `document.getElementById(id)` for hardcoded module-level SECTION_IDS only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | `locale` island prop | accept | Value originates server-side from `Astro.currentLocale`, constrained to the `en`/`es` allowlist by astro:i18n `prefixDefaultLocale` config (Phase 21); not derived from any runtime user input. `translations[locale]` indexes a static build-time object. |
| T-25-02 | Injection | `scrollToId(id)` DOM lookup | accept | `id` is always drawn from the hardcoded module-level `SECTION_IDS` array — never from URL, query, hash, or user input — so no attacker-controlled selector reaches `getElementById`. |
| T-25-03 | Denial of Service | scroll/resize listeners | mitigate | Handlers are `requestAnimationFrame`-throttled via `tickingRef` (ported verbatim, D-03) and registered `{ passive: true }`; `client:visible` further defers all listener registration until the element scrolls into view. |
| T-25-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase — the port reuses only existing deps (React, `translations`, `useActiveSection`). No RESEARCH.md Package Legitimacy Audit required; nothing added to package.json. |
</threat_model>
