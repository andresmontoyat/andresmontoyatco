Modify the existing `[lang]` `useEffect` (currently lines 21–30) — keep the existing logic for `<html lang>`, `<title>`, `<meta name="description">`, and ADD a `setMeta` helper plus 4 setMeta calls for og:title, og:description, twitter:title, twitter:description.

    Final useEffect block:

    ```jsx
    useEffect(() => {
      if (typeof document === 'undefined') return
      document.documentElement.lang = lang
      const meta = translations[lang] && translations[lang].meta
      if (!meta) return

      if (meta.title) document.title = meta.title
      const descTag = document.querySelector('meta[name="description"]')
      if (descTag && meta.description) descTag.setAttribute('content', meta.description)

      const setMeta = (selector, content) => {
        const el = document.querySelector(selector)
        if (el && content) el.setAttribute('content', content)
      }
      setMeta('meta[property="og:title"]', meta.title)
      setMeta('meta[property="og:description"]', meta.description)
      setMeta('meta[name="twitter:title"]', meta.title)
      setMeta('meta[name="twitter:description"]', meta.description)
    }, [lang])
    ```

    Notes (verbatim from UI-SPEC lines 860–875):
    - Reuses the existing `translations[lang].meta.title` / `meta.description` (added in Phase 2 — no new translation keys needed for this plan).
    - The `setMeta` helper is internal to the effect (no module-level export; keep the API surface clean).
    - Each setMeta call is idempotent: if the meta tag is missing from the DOM (e.g., not added to index.html in Task 1), the helper silently no-ops (`if (el && content)`).
    - The early return after `if (!meta) return` is a small refactor (the original code uses `if (meta) { ... }`); both are equivalent. Pick the early-return shape per the UI-SPEC pattern. Make sure the meta object accessor `translations[lang] && translations[lang].meta` is preserved.

    - Imports or exports
