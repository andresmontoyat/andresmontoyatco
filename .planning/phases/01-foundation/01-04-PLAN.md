  <action>
**Step 1 — Append reduced-motion rules to src/index.css:**

Add the following block AFTER the existing `@layer utilities { }` block:

```css
/* ── Accessibility: respect OS Reduce Motion preference ── */
/* DSGN-03: This rule is the global baseline suppression layer.
   Component-level animations additionally use the Tailwind
   motion-safe: variant prefix (e.g., motion-safe:animate-slide-up)
   so they are suppressed at the utility level too. */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This rule uses `!important` intentionally — it must override inline styles, Tailwind utilities, and any JS animation library that applies styles directly. This matches the WCAG 2.1 recommended pattern.

**Step 2 — Wrap the `pulse2` animation in tailwind.config.js with motion-safe context note:**

The `pulse2` animation runs `infinite` and is currently used on the hero status indicator. In Phase 2, it must be applied as `motion-safe:animate-pulse2` (not `animate-pulse2`) so it automatically suppresses when reduced motion is enabled.

Add a comment in tailwind.config.js above the `pulse2` keyframe to make this requirement explicit for Phase 2 implementors:

```js
// NOTE: pulse2 runs infinite — MUST be applied with motion-safe: prefix
// in all components: use 'motion-safe:animate-pulse2' never 'animate-pulse2'
pulse2: {
  '0%,100%': { opacity: '1' },
  '50%':     { opacity: '0.4' },
},
```

**Step 3 — Verify the rule works locally (manual step for executor):**
Open `npm run dev` in browser, open DevTools > Rendering tab > check "Emulate CSS media feature prefers-reduced-motion" to "reduce". Scroll to any animated element. Confirm animations do not play.

If DevTools Rendering tab is not available, use: `@media (prefers-reduced-motion: reduce)` check in CSS — the rule is valid and will be tested by the Phase 1 verification checkpoint.
  </action>
