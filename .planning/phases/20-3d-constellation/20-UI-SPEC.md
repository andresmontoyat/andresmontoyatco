# Phase 20 — UI Design Contract

> Visual and interaction contract for the v3.10 3D constellation upgrade. Generated by gsd-ui-researcher 2026-06-08, verified by gsd-ui-checker.
>
> **Scope reminder:** This UI-SPEC covers the surfaces NEW to Phase 20 — the 3D camera/depth visual contract, OrbitControls gesture affordances, the bilingual onboarding hint pill, prefers-reduced-motion path divergence, keyboard nav non-extension, the ErrorBoundary fallback continuity, and a11y surface. Existing tokens (`ink-*`, `brand`, `accent`, `text-*`, `constellation-*`, `hintPill-*`), animations (`motion-safe:animate-pulse2 / animate-fadein / animate-svg-twinkle`), and the bilingual `t.*` namespace are REUSED — Phase 20 introduces ONE new keyframe family (`hintPillFadeIn`) and TWO new i18n keys (`t.game.hint.drag.{en,es}`).

| Destructive confirmation | n/a — Phase 20 has zero destructive actions | n/a | n/a |

**Hint pill copy rationale:**

- Two words (EN) / three words (ES) — short enough to read in <500 ms peripheral vision.
- Imperative verb + adverb ("drag" / "rotate" / "arrastra" / "rotar") — matches Phase 15 hint pill voice ("Click a star" / "Toca una estrella") for tonal consistency.
- Lowercase — matches the Phase 15 precedent. NO sentence-case, NO trailing period.
- NO icon — pill is text-only; cursor `grab` state is the visual icon, the pill is the verbal label. This matches Phase 15 (no icon there either).
- NO bilingual concatenation in a single pill (`"drag to rotate · arrastra para rotar"` style of `t.game.hintPill`) — the OnboardingHint is shown ONCE per visit (localStorage `cam-3d-hint-seen`), so concatenation would double its length needlessly. The user already chose a language via `cam-lang`; respect it.

**Reasoning vs Phase 15 precedent:** Phase 15 `t.game.hintPill` ("Click a star · Toca una estrella") concatenates EN+ES because it shows under `prefers-reduced-motion + !hasInteracted` indefinitely (until first interaction). Phase 20's hint pill auto-dismisses at 5s + on first drag, so it's seen exactly once per visit (or zero times after localStorage flag set) — single-language is correct.

**`t.game.hint.drag` namespace shape (Plan 20-02 i18n edit, exact JSON):**

```js
// src/i18n/translations.js — both en{} and es{} branches
game: {
  // ... existing keys preserved ...
  hint: {
    drag: 'drag to rotate', // ES branch: 'arrastra para rotar'
  },
}
```

The nested `hint.drag` namespace leaves room for v3.10.1+ future hint additions (`hint.zoom`, `hint.reset`) without polluting `t.game.*` with N flat keys.

### Camera

| Property | Value | Rationale |

### Edge depth handling (MOD-04 mitigation)

| Property | Value | Rationale |
|----------|-------|-----------|
| Edge material | Custom `ShaderMaterial` extending existing edge shader from Phase 17 D-17-EDGE-RGBA | Preserves per-vertex alpha contract |
| Per-vertex alpha falloff | `gl_FragColor.a *= 1.2 - abs(normalView.z)` (in fragment shader, FEATURES MOD-04 Option A) | Edges going "into the screen" fade toward transparent; edges in the camera plane stay full opacity. Adds 0 kB (shader-only). REJECTS Line2 (+10 kB over budget). |
| Edge `depthWrite` | `false` (MOD-02 z-fighting mitigation) | Edges never compete with nodes for the z-buffer. Render edges FIRST in the scene order (Phase 17 order preserved at `WebGLConstellation.js:411-412`). |

### Node depth handling (CRIT-05 mitigation)

| Property | Value | Rationale |
|----------|-------|-----------|
| Vertex shader size-attenuation | `gl_PointSize *= perspectiveScale / -mvPosition.z` where `perspectiveScale = uCanvasHeight / (2.0 * tan(uFovRad / 2.0))` | Near nodes grow proportionally, far nodes shrink proportionally — primal depth cue. CRITICAL — without this the scene reads broken. |
| `gl_PointSize` clamp | `clamp(..., 1.0, 64.0)` | Defends against extreme near-plane magnification on aggressive rotation. |
| New uniforms | `uCanvasHeight` (px), `uFovRad` (radians) — both updated on resize | Required for the size-attenuation formula. |
| Existing uniforms preserved | `uTime`, `uHaloPulse`, halo/flash uniforms from Phase 17 Slice 4 | Phase 20 EXTENDS the shader, does NOT replace it. |

### Auto-rotate cadence & resume policy (D-20-CONTEXT-AUTOROTATE-RESUME)

| Event | Behavior | Notes |
|-------|----------|-------|
| Frame 0 (component mount) | `autoRotate = true` IF `!prefersReducedMotion.matches` | Defensive RM gate inside renderer (CRIT-06 belt-and-braces). |
| Hover registers on any node (`hoveredSkillId != null`) | `autoRotate = false` (paused) | MIN-01 — recruiter examining a node shouldn't fight rotation. |
| Hover leaves (`hoveredSkillId === null`) | `autoRotate = true` IF first drag has NOT happened yet | Allows the "I peeked at a node, now I want to see the spin again" case BEFORE first interaction. |
| ExperienceCard open (`selectedSkillId != null`) | `autoRotate = false` | MIN-01 — card open = focused exploration mode. |
| ExperienceCard close (`selectedSkillId === null`) | `autoRotate = true` IF first drag has NOT happened yet | Same as hover-leave above. |
| First `controls 'start'` event (first drag begins) | `autoRotate = false` PERMANENTLY (page-session-scoped) | D-20-CONTEXT-AUTOROTATE-RESUME — recruiter took the wheel; don't yank it back. |
| Reload page | `autoRotate = true` again at mount | New page session = restart cycle. |
| Spin direction | Counter-clockwise (CCW) from camera's initial view (positive Y rotation in three.js) | Three.js default. No override. |
| Spin axis | World Y (vertical) | Three.js default. NEVER X or Z (vestibular nausea, FEATURES anti-feature). |
| Cadence | Constant linear rotation (no ease, no acceleration) | Three.js `autoRotateSpeed` is constant. Constant ease = predictable, recruiter-readable. Easing in/out would feel "trying too hard." |

## ErrorBoundary Visual Continuity Contract

When the WebGL render path fails (shader compile, OOM, or `webglcontextlost`), the recruiter MUST see a smooth degradation, not a broken-UI flash. This is the "no flash, no flicker, recruiter cannot tell something failed" contract.

| Failure mode | What recruiter sees | What happens technically |
|--------------|---------------------|--------------------------|
| Shader compile error at mount | The WebGL canvas never appears; SVG `<svg viewBox="0 0 1000 1000">` renders in the same flex slot immediately | `RendererErrorBoundary` (Phase 17) catches; falls back to `SvgConstellation`. No error banner. |
| `webglcontextlost` after mount (tab return) | The WebGL canvas content disappears for ~1 frame, then SVG mounts in the same slot OR three.js auto-restores | D-20-CONTEXT-LOSS handlers: (a) call `e.preventDefault()`; (b) emit `onContextLost?.()` callback → `GameMode` swaps to SVG via existing capability hook. |
| `webglcontextrestored` | If still mounted, three.js rebuilds geometry; if swapped to SVG, stays on SVG | Defensive: prefer to STAY on SVG once swapped. Re-mounting WebGL after a context-loss restore introduces re-mount flicker. |
| Runtime exception during render | RendererErrorBoundary catches; SVG fallback renders in the same slot | Phase 17 pattern preserved. |
| Hint pill present at failure moment | Hint pill unmounts cleanly when WebGL unmounts (component lives in `GameMode.js`, unmounts when `WebGLConstellation` unmounts) | No zombie pill over SVG path. |

**Recruiter-visible continuity rule:** The flex slot dimensions DO NOT shrink/grow during fallback. SVG `<svg>` uses `width="100%"` + `style={{minHeight: 240, maxHeight: 600}}` (Phase 15) — same envelope as the WebGL canvas. ZERO layout shift.

## State Transitions — Visual Narrative (frame-by-frame for executor)

| Moment | What renders | What animates | Recruiter perception |
|--------|--------------|---------------|----------------------|
| Page load, before WebGL chunk arrives | SVG path renders eagerly (Phase 14 fallback) | SVG node-reveal stagger (Phase 15) + SVG twinkle (Phase 19) | "There's a constellation here, it's alive" |
| WebGL chunk arrives + mounts | Canvas swaps in over SVG (cross-fade if Phase 17 has it, else hard swap) at tilted ~15°/~10° initial angle | Auto-rotate begins immediately at 0.5 speed (CCW around Y) | "Wait, it's 3D" — depth read at frame 0 via initial tilt; auto-rotate confirms over the next 500 ms |
| 800 ms after WebGL mount | Hint pill fades in (`animate-fade-in`, 500 ms) below canvas | Pill opacity 0 → 1 | "drag to rotate" — recruiter reads the verb |
| Recruiter mousemoves over canvas | Cursor becomes `grab` | None | "I can grab this" |
| Recruiter mousemoves over a node | Cursor becomes `pointer`; hover edge-reveal (weight-1 edges) fades in | Edge alpha 0 → 1 (200 ms) | "This node has connections" |
| Recruiter pointerdown | Cursor becomes `grabbing`; auto-rotate pauses; if pointerup arrives within 5px+250ms → click-confirmed → ExperienceCard opens | None during pointerdown; ExperienceCard `cardFadeIn` on click confirm | "Held = grab; quick tap = open" |
| Recruiter drags (pointerdown → 5px+) | Camera rotates via OrbitControls with damping; hint pill fades out (`animate-hint-fade-out`, 600 ms); localStorage `cam-3d-hint-seen = 'true'`; auto-rotate permanently paused | Camera position interpolates with damping | "I control this" |
| Recruiter releases mid-drag | Camera coasts to stop via damping (settles <1s) | Damped angular velocity decay | "It has weight" |
| Recruiter drags past polar clamp | Camera halts at clamp angle; no snap-back, no visual indicator | Camera y-rotation hits ceiling | Soft resistance felt via damping, not seen as a wall |
| 5 s pass without any drag | Hint pill auto-dismisses (`animate-hint-fade-out`); auto-rotate continues | Pill opacity 1 → 0 | Pill quietly retires; constellation keeps spinning |
| Recruiter clicks a node (confirmed click) | ExperienceCard `cardFadeIn` (Phase 16); auto-rotate pauses (selectedSkillId != null); node halo brightens (Phase 17 shader) | Card 200 ms scale 0.96 → 1.0 + opacity 0 → 1 | "I selected this; here's what I did with it" |
| Recruiter closes ExperienceCard | Card unmounts; auto-rotate resumes IF first drag has NOT happened | None | Return to ambient state |
| Recruiter switches tab + returns | If context survives: rAF resumes from `visibilitychange` (Phase 17) | None | Continuity |
| Recruiter switches tab + returns + context lost | `webglcontextlost` → SVG path swaps in silently | SVG node-reveal stagger replays | Recruiter sees 2D, but constellation still there. No error. |
