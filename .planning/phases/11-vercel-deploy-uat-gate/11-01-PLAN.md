<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local working copy → committed repo | `.vercel/` (project ID + auth token) must NOT cross this boundary |
| Repo → Vercel build pipeline | `vercel.json` is the contract; misconfiguration leaks infra paths or breaks cache strategy |
| Vercel edge → browser/share-bot | OG meta tags determine what share cards render (LinkedIn, Twitter, Slack preview) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01-01 | I (Info Disclosure) | `.vercel/` directory | mitigate | Task 2 adds `.vercel/` to `.gitignore` BEFORE Plan 11-04 runs `vercel link`. CLI writes project ID + auth token to this dir; if committed, anyone with repo read access can hijack the Vercel project. |
| T-11-01-02 | T (Tampering) | `vercel.json` cache header rule for `/index.html` | mitigate | Explicit `max-age=0, must-revalidate` on `/index.html` (Task 1) prevents intermediary caches serving stale shell after deploy. Stale index = client loads old asset hashes that 404 against new deploy. |
| T-11-01-03 | S (Spoofing) | OG meta `og:url`, `og:image`, `twitter:image` | mitigate | Task 3 corrects domain typo so share-bots see consistent canonical. Mismatched domain in OG tags causes LinkedIn/Twitter to either drop the card or display unverified-domain warnings. Phase 12 will land matching live domain. |
| T-11-01-04 | I (Info Disclosure) | SPA rewrite `/(.*)` → `/index.html` | accept | Universal rewrite is safe because portfolio has zero `/api/` routes and Vercel handles `/_next/` style infra paths internally before rewrites apply. No path-traversal surface to leak. |
| T-11-01-05 | D (DoS) | `/assets/(.*)` `max-age=31536000 immutable` | accept | Long-cache on hashed bundles is the goal; "DoS" by stale cache is impossible because Vite hashes filename per content change. Worst case: user keeps old bundle until they reload — which is the intended caching contract. |

All threats LOW severity. No HIGH severity — Phase 11 is deploy-config only (no auth, no user data, no PII surface).
</threat_model>
