<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL fragment → island DOM | `window.location.hash` is attacker-controllable (user-supplied URL) and is concatenated into the LangPill `<a href>` |
| client → persistence | LangPill writes `cam-lang` cookie; ThemeToggle writes `cam-theme` localStorage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Tampering (XSS) | LangPill href = locale-prefix + `window.location.hash` | mitigate | Hash is appended to a hardcoded same-origin locale-prefix path and assigned via React's `href` attribute (React auto-escapes attribute values). Never rendered with `dangerouslySetInnerHTML`, never used as a JS scheme; the `/en/`|`/es/` prefix fixes scheme+origin so a hostile hash cannot redirect off-site or inject markup. |
| T-22-02 | Elevation (open-redirect) | Language switcher target | mitigate | Target locale is a hardcoded literal `'en'`/`'es'` allowlist; hrefs are built by `getRelativeLocaleUrl(literal, path)` in server frontmatter, never from user input — conceptually mirrors middleware.ts `isKnownLocale`. |
| T-22-03 | Information disclosure | `cam-lang` cookie / `cam-theme` localStorage | accept | Contains only a locale/theme preference, no PII or auth material; low-value target. `path=/`, no HttpOnly needed (client must read/write). |
| T-22-N/A | Tampering (supply chain) | npm/pip/cargo installs | n/a | No package installs in this phase — ports existing in-repo code only; no new dependencies added. |
</threat_model>
