## v3.8 Requirements — All Delivered

Interactive "game mode" — a skill constellation as the default landing, with a persisted toggle to the
current ("dev") view. Phase numbering continues from v3.7 → **starts at Phase 14**. Adaptive render
(WebGL desktop / SVG-DOM mobile) holds the Lighthouse mobile gate, WCAG 2.1 AA, and SEO.

### Game Mode

- [x] **GAME-01** Visitor lands on an interactive skill constellation by default; the render adapts to the
  device — full WebGL on desktop, a lightweight SVG/DOM static path on mobile (and on no-WebGL,
  reduced-motion, or save-data). Same content in both; capability + viewport + `prefers-reduced-motion`
  detection chooses the renderer behind a single props contract. *(delivered Phase 15 SVG + Phase 17 WebGL)*
- [x] **GAME-02** Constellation nodes are skills (Carlos's technologies); edges connect skills that
  co-occur in the same job; nodes cluster visually by skill category (color/zone). Graph derived from
  `experience.js` `tech[]` + a new `skills.js` category map (with GCP/Google Cloud normalization). *(delivered Phase 14 derivation + Phase 15 render)*
- [x] **GAME-03** Visitor can filter/navigate by selecting multiple skills, by a year/timeline range
  (2007–2026), and by skill category; a reset clears all filters. Filters are combinable (intersection)
  and computed by pure selectors. *(delivered Phase 16, 2026-06-03)*
- [x] **GAME-04** Selecting a skill shows the experiences where Carlos used it as floating bilingual
  cards — title, company, date, location, bullets, tech chips (colored by category) — plus a CV CTA. *(delivered Phase 16, 2026-06-03)*
- [x] **GAME-05** Visitor can toggle between game mode and the current dev view with one click; game mode
  is the default landing; the choice persists in `localStorage` (`cam-viewmode`) and is addressable via a
  `?mode=` deep-link. *(delivered Phase 14)*
- [x] **GAME-06** Game mode is accessible and crawlable: skill nodes are keyboard-navigable with
  `aria-label`s, experience cards use dialog/popover semantics (focus trap, Esc to close), a visually-hidden
  full-experience list is present in the DOM for screen readers / crawlers / ATS in both render paths, and
  `prefers-reduced-motion` disables physics/particles. WCAG 2.1 AA contrast holds in dark and light themes. *(delivered Phase 15)*
- [x] **GAME-07** Game mode holds the Lighthouse mobile HARD gate (Perf ≥95 / A11y 100 / BP 100 / SEO 100,
  matching the v3.4 baseline): three.js loads only in the lazy desktop chunk, layout positions are baked at
  build time (no d3-force in the client), and the mobile path adds < ~30KB gz over the current baseline. *(delivered Phase 17, 2026-06-06; mobile chunk 8.91 kB gz; Lighthouse HARD gate cleared)*
