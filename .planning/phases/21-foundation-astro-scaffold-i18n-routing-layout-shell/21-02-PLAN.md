Build `BaseLayout.astro` — the shared `<head>` shell for every page — and the symmetric `/en` + `/es` route tree that consumes it. BaseLayout ports the existing `index.html` head (favicon, viewport, theme-color, hero preload, OG/Twitter, JSON-LD Person, GA) and adds three build-time behaviors the current React site does at runtime: per-locale `<html lang>` (ROUTE-04), per-locale `<title>`/description + hreflang/canonical (ROUTE-03), and a FOUC-free blocking theme-flip inline script (ISLAND-04).

Purpose: Every visible page in the migration renders through this layout. Getting `<html lang>`, hreflang/canonical, and the theme-flip right at build time is what makes `/en` and `/es` real static SEO-correct routes (ROUTE-01) instead of a client-rendered SPA.
Output: `BaseLayout.astro` + three thin locale route files whose built HTML passes lang/hreflang/canonical/theme-script assertions.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time env → HTML | `PUBLIC_SITE_URL` is interpolated into canonical/hreflang/og:url at build time |
| browser localStorage → inline script | The blocking `<head>` script reads `cam-theme` from client storage before paint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-02-01 | Tampering | inline theme `<script is:inline>` scope creep | mitigate | Keep the inline block scoped to theme-flip only; no cookie/analytics/redirect logic added to it (per 21-RESEARCH.md Security Domain) |
| T-21-02-02 | Denial of Service | inline script throwing in private-mode Safari before paint | mitigate | Wrap `localStorage` access in `try/catch` (mirrors ThemeContext) so a storage exception cannot abort render before first paint |
| T-21-02-03 | Information Disclosure | `PUBLIC_SITE_URL` in client HTML | accept | Value is the public canonical domain (`https://andresmontoyat.co`) — non-sensitive by design (D-05); it is meant to be public in canonical/OG tags |
| T-21-02-04 | Spoofing | canonical/hreflang built from `Astro.url.pathname` | mitigate | Locale prefixes come from Astro's own build-time route tree (`en`/`es` only), not user input; path stripped via a fixed `/^\/(en|es)/` regex, no reflected user value enters the URL |
</threat_model>
