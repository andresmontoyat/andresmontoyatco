## Decisions

### Milestone-level (logged at v3.8 open, 2026-05-29)

- **D-v3.8-SCOPE** — feature inside existing portfolio repo; new milestone, NOT new-project.
- **D-v3.8-METAPHOR** — skill constellation; node = skill (job is secondary; click skill → experiences as cards).
- **D-v3.8-NATURE** — playable interactive visualization — no objective, no win/lose.
- **D-v3.8-DEFAULT** — game mode is default landing; toggle to dev view persisted.
- **D-v3.8-ADAPTIVE** — WebGL desktop / SVG-DOM mobile resolves the conflict between game-default and the HARD Lighthouse mobile gate.
- **D-v3.8-TEST** — introduce Vitest + RTL this milestone (test-infra debt paydown after 9 deferred milestones).
- **D-v3.8-PHASES** — phase numbering continues → v3.8 starts at Phase 14.
- **D-v3.8-ORDER** — Light/SVG path (Phase 15) + filters/cards (Phase 16) ship BEFORE WebGL (Phase 17), so Lighthouse gate is verified against the gate-safe baseline first.

### Plan-level highlights

- **D-14-01-LAYOUT** — deterministic radial category-centroid layout chosen over d3-force (zero new deps, deterministic, no client-side physics).
- **D-15-VIS-EDGE** — weight-1 edges revealed on hover/select only (not always-visible; reduces visual noise).
- **D-15-KB-ACTIVATE** — roving tabindex pattern for spatial navigation (Enter activates, Esc deselects).
- **D-16-INTERSECT-AND** — multi-skill filter is AND semantics (intersection) — strongest recruiter signal.
- **D-16-PERSIST-MEMORY** — filter state in-memory only; no localStorage write (lands fresh on reload).
- **D-16-BUNDLE-GATE** — programmatic gzip-size enforcement with HARD/WARN/PASS bands (mirrors Phase 16 pattern in Phase 17).
- **D-17-LIB** — three.js raw (~70 kB gz target ~40 kB tree-shaken; actual 117 kB gz desktop lazy chunk). Reconsidered force-graph after flagging Phase 14 D-14-01-LAYOUT conflict (force-graph's d3-force fights baked positions).
- **D-17-HOVERED-PROP** — `hoveredSkillId` owned by useConstellation; WebGL consumes via props (preserves SC-1 props-contract parity between SVG + WebGL renderers).
- **D-17-EDGE-RGBA** — `BufferAttribute(arr, 4)` + `material.transparent=true` (avoids light-theme parity failure where pre-multiplied RGB fades to black against light bg).
- **D-17-LIGHTHOUSE-FLOW** — Phase 17 close runs `npm run lighthouse:mobile && npm run lighthouse:check` (existing scripts from Phase 11-02). Local verdict only; Plan 11-05 deployed-vercel-domain gate stays deferred.
