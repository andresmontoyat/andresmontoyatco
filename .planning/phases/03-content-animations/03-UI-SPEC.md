### useInView Hook (ANIM-02)

**File:** `src/hooks/useInView.js`

**Modeled after:** `src/hooks/useActiveSection.js` (same guard pattern, same dep-string trick).

**Signature:**
```js
// useInView(ref, options) → boolean
// ref: React ref attached to the element to observe
// options.threshold: default 0.25
// Returns true once element has entered viewport; stays true (unobserved after first fire)
export default function useInView(ref, { threshold = 0.25 } = {})
```

**Implementation spec:**
- Guard: `if (typeof window === 'undefined' || typeof IntersectionObserver === 'undefined') return true` — SSR safe; returns true so content is visible without JS.
- Observer config: `root: null`, `rootMargin: '0px'`, `threshold` (from options, default 0.25).
- On intersection (`entry.isIntersecting === true`): set state to `true`, then call `observer.unobserve(entry.target)` — fires once per session, never replays on scroll-back.
- Cleanup: `observer.disconnect()` on unmount.
- Returns: `boolean` — `false` before element is in view, `true` after first intersection.

**CSS class pattern (used by all animated elements):**

Add to `src/index.css` under `@layer utilities`:
```css
.animate-on-scroll {
  opacity: 0;
  transform: translateY(16px);
  transition: opacity 700ms ease-out, transform 700ms ease-out;
}

.animate-on-scroll.is-visible {
  opacity: 1;
  transform: translateY(0);
}

@media (prefers-reduced-motion: reduce) {
  .animate-on-scroll {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

**Usage pattern in components:**
```jsx
const ref = useRef(null)
const inView = useInView(ref, { threshold: 0.25 })

<div
  ref={ref}
  className={`animate-on-scroll${inView ? ' is-visible' : ''}`}
  style={{ transitionDelay: `${i * 100}ms` }}
>
```

Note: Use `transitionDelay` (not `animationDelay`) because `.animate-on-scroll` uses CSS `transition`, not `animation`. The delay causes staggered reveal as each item transitions when the parent container enters view.

**Reduced motion:** The `@media (prefers-reduced-motion: reduce)` block in `.animate-on-scroll` sets opacity:1 and transform:none unconditionally — content is immediately visible with no transition. This is additive to the existing global rule in src/index.css (which collapses all transition durations). Both layers provide protection.

**EmailHeroCard spec:**

```jsx
function EmailHeroCard({ className }) {
  const [copied, setCopied] = useState(false)
  const email = 'andresmontoyat@gmail.com'

  function handleCopy() {
    navigator.clipboard.writeText(email).then(() => {
      setCopied(true)
      setTimeout(() => setCopied(false), 1500)
    }).catch(() => {})   // silent fail per copywriting contract
  }

  return (
    <a
      href={`mailto:${email}`}
      className={`block bg-ink-500 border border-ink-400 rounded-xl p-6
                  hover:border-brand hover:-translate-y-1 transition-all duration-200
                  relative ${className}`}
    >
      {/* Label */}
      <div className="font-mono text-xs text-text-muted uppercase tracking-wider mb-2">
        {t.contact.email}
      </div>

      {/* Email + copy button row */}
      <div className="flex items-center justify-between gap-4 flex-wrap">
        <div className="text-xl sm:text-2xl font-extrabold text-text-primary break-all">
          {email}
        </div>
        <button
          onClick={(e) => { e.preventDefault(); handleCopy() }}
          aria-label="Copy email to clipboard"
          className="flex-shrink-0 font-mono text-xs text-brand
                     border border-brand/30 hover:border-brand
                     rounded-full px-3 py-1.5 transition-colors duration-150
                     min-h-[32px] min-w-[80px] text-center"
        >
          <span aria-live="polite">
            {copied ? 'Copied!' : 'Copy email'}
          </span>
        </button>
      </div>

      {/* Subtle brand glow on hover to reinforce hero status */}
      <div className="absolute inset-0 rounded-xl bg-brand/5 opacity-0
                      group-hover:opacity-100 pointer-events-none transition-opacity duration-200" />
    </a>
  )
}
```

Key interaction points:
- `e.preventDefault()` on copy button click prevents the `mailto:` anchor from firing when copy button is clicked
- `aria-live="polite"` on the label span — screen readers announce "Copied!" on state change without interrupting
- Copy button has `min-h-[32px]` (not 44px) — it is a secondary action inside a card; the primary tap target is the full card `<a>`. If Phase 4 audits require 44px, pad externally.
- `href="mailto:..."` on the card anchor — clicking anywhere on the card (except the copy button) opens the mail client
