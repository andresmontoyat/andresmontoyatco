    function useCharReveal(target, startOffsetMs, perCharMs) {
      const [text, setText] = useState('')
      const timersRef = useRef([])

        timersRef.current = []
        setText('')

        if (typeof window === 'undefined') return undefined

            setText(target.slice(0, i))
          }, startOffsetMs + i * perCharMs)
          timersRef.current.push(id)
        }

        return () => {
          timersRef.current.forEach(clearTimeout)
          timersRef.current = []
        }

    Critical implementation rules from UI-SPEC and CONTEXT:

    1. **Stagger sequence (UI-SPEC table lines 197-208):**
       - badge: delay 0ms, fade-in 500ms
       - h1a: delay 150ms, slide-up 600ms
       - h1b: delay 300ms, fade-in 500ms (then internal char reveal starts at 300ms + 40ms × N)
       - h1c: delay 500ms, slide-up 600ms
       - lead: delay 650ms, slide-up 600ms
       - CTA row: delay 800ms, slide-up 600ms
       - stats: delay 950ms, fade-in 500ms
       Each delay/duration MUST match these values exactly.

    2. **Animation pattern**: every animated element MUST use the exact pair `motion-safe:opacity-0 motion-safe:animate-{fade-in|slide-up}` plus `style={{ animationDelay: 'Xms', animationFillMode: 'forwards', animationDuration: 'Yms' }}`.
       - The `motion-safe:` prefix means under prefers-reduced-motion the `opacity-0` and `animate-*` utilities are NOT applied — the element renders fully opaque immediately. This is exactly the DSGN-03 layered defense.
       - `animationFillMode: 'forwards'` keeps the final state.
       - `animationDuration` overrides the Tailwind default (0.5s/0.6s) where UI-SPEC pins a specific value.

    3. **Char-reveal mechanism (UI-SPEC lines 218-241):**
       - Custom `useCharReveal(target, startOffsetMs, perCharMs)` hook (defined inline in Hero.js — keeps the file self-contained).
       - 40ms per char; 300ms start offset (matches stagger step 3 visible-at).
       - Resets on `target` change (so language switch restarts the reveal cleanly).
       - Stores timeout ids in a `useRef` array for cleanup on unmount and on dependency change.
       - Reduced-motion guard: `window.matchMedia('(prefers-reduced-motion: reduce)').matches` → set `text = target` immediately, no setTimeouts.
       - SSR-safe: returns early if `typeof window === 'undefined'`.

    4. **Cursor**: blinking `|` character via `motion-safe:animate-pulse2`. Removed when `displayText.length === target.length`. The cursor span uses `text-text-primary` color — explicitly NOT the gradient (cursor sits visually in the gradient line but should stay legible).
