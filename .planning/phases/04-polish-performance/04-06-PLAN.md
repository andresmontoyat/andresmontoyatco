API:
```sh
npx lighthouse <URL> \
  --preset=mobile \
  --throttling-method=simulate \
  --output=json \
  --output-path=lighthouse-report.json \
  --quiet \
  --chrome-flags="--headless --no-sandbox --disable-gpu"
```

Score extraction (categories): values are 0-1 (multiply ×100 for the visible score).
```sh
cat lighthouse-report.json | jq '.categories.performance.score'
cat lighthouse-report.json | jq '.categories.accessibility.score'
cat lighthouse-report.json | jq '.audits."tap-targets".score'
```

- `tap-targets` — D-05 + D-06 acceptance criterion (must be score: 1)

- `first-contentful-paint` — &lt; 1.8s on mobile preset for green
- `unused-css-rules` — informational; score is informational
- `render-blocking-resources` — should be 0 with Vite (CSS is inlined)

Vite preview server:

Lighthouse can detect a system Chrome via `--chrome-flags`, OR use the Playwright-installed binary by setting `CHROME_PATH` env var. For agent-friendliness, the script below uses the headless flag so Lighthouse will spawn whatever Chrome it finds.

    **Step 2 — Add scripts to package.json.** Add three scripts to the `"scripts"` block:
    ```json

    "lighthouse:mobile": "npm run build && (npx vite preview --port=4173 &) && sleep 3 && npx lighthouse http://localhost:4173 --preset=mobile --output=json --output=html --output-path=./lighthouse-report-mobile --quiet --chrome-flags=\"--headless --no-sandbox --disable-gpu\" && pkill -f 'vite preview' || true",
    "lighthouse:check": "node -e \"const r = require('./lighthouse-report-mobile.report.json'); const p = r.categories.performance.score; const a = r.categories.accessibility.score; const t = r.audits['tap-targets'].score; console.log('Performance:', p, 'A11y:', a, 'Tap-targets:', t); if (p &lt; 0.9 || a &lt; 0.9 || t !== 1) process.exit(1);\""
    ```

    Insert after the existing `build:analyze` script. Final scripts block (relative ordering):
    ```json
    "scripts": {
    ```

    - `pkill -f 'vite preview'` — clean shutdown of the backgrounded server
    - `|| true` — swallow pkill failure on first run (no preview process exists)
    - The `lighthouse:check` script is a programmatic gate: exits non-zero if any threshold fails. Plan 04-06 Task 2 invokes this for automated acceptance.

    No semicolons in package.json (it's JSON). Per JSON syntax — trailing commas NOT allowed. Match existing style.
