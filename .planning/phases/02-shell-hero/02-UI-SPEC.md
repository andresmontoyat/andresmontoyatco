### Hero Entrance Animation Sequence (HERO-02)

**Decision (user):** Staggered entrance animations play automatically on load.

**Mechanism:** CSS animation-delay stagger. Each element starts with `opacity: 0` via an `animate-fade-in` or `animate-slide-up` class applied with a delay utility. Use `motion-safe:` prefix on all animation classes (DSGN-03 compliance).

**Stagger sequence and timing:**

| Step | Element | Animation | Delay | Duration | Total visible at |
|------|---------|-----------|-------|----------|-----------------|
| 1 | Status badge | fade-in | 0ms | 500ms | 500ms |
| 2 | h1a (plain line) | slide-up | 150ms | 600ms | 750ms |
| 3 | h1b (gradient line — char-reveal) | char-reveal (see below) | 300ms | ~800ms | 1100ms |
| 4 | h1c (plain line) | slide-up | 500ms | 600ms | 1100ms |
| 5 | Lead paragraph | slide-up | 650ms | 600ms | 1250ms |
| 6 | CTA button row | slide-up | 800ms | 600ms | 1400ms |
| 7 | Stats grid | fade-in | 950ms | 500ms | 1450ms |

All delays implemented via inline `style={{ animationDelay: 'Xms' }}` on each wrapper element. No third-party library.

**Reduced motion:** All `motion-safe:animate-*` classes are already suppressed by the global `prefers-reduced-motion` rule in src/index.css. Elements must be visible (opacity: 1) without animation when reduced-motion is active — set a `motion-reduce:opacity-100` class or initialize opacity to 1 via a wrapper that only applies opacity-0 under `motion-safe:`.

**Pattern for each animated element:**
```
className="motion-safe:opacity-0 motion-safe:animate-slide-up"
style={{ animationFillMode: 'forwards', animationDelay: 'Xms' }}
```

### Hero h1b Character Reveal (HERO-04)

**Decision (user):** Gradient line h1b only — animate just the gradient line ("backend systems" / "backend escalables").

**Mechanism:** React `useState` with `useEffect` + `setTimeout` per character. No CSS-only solution because the gradient `bg-clip-text` must apply to the full string; we reveal characters by appending to a `displayText` state string.

**Specification:**

- Component: local state `const [displayText, setDisplayText] = useState('')`
- On mount (and on `lang` change): reset `displayText` to `''`, then schedule one `setTimeout` per character of `t.hero.h1b`
- Delay per character: 40ms (i.e., character N appears at `startOffset + N * 40ms`)
- Start offset: 300ms (matches stagger table step 3 above — the element becomes visible at 300ms, then chars start)
- Total reveal duration for "backend systems" (15 chars): 300ms + 15 * 40ms = 900ms ✓
- Total reveal duration for "backend escalables" (18 chars): 300ms + 18 * 40ms = 1020ms ✓
- Cursor: a blinking `|` appended while reveal is in progress; removed when `displayText.length === t.hero.h1b.length`
- Cursor blink: `motion-safe:animate-pulse2` on a `<span>` wrapping the cursor char
- Cleanup: clear all pending timeouts on unmount and on `lang` change (store timeout IDs in a ref array)
- The outer `<span>` carrying `bg-brand-gradient bg-clip-text text-transparent` wraps `{displayText}` — gradient applies to whatever characters are visible
- Accessibility: the full `t.hero.h1b` string is provided via `aria-label` on the containing `<h1>` so screen readers announce the complete headline immediately without waiting for the reveal

**Implementation guard:** Wrap the entire char-by-char effect in `if (window.matchMedia('(prefers-reduced-motion: reduce)').matches)` — if true, set `displayText` to the full string immediately without scheduling timeouts.

### Mobile Menu Overlay (NAV-01)

**Decision (user):** Fade in — opacity 0 to 1 over 200ms. Body scroll-locked behind it. ESC and close tap both dismiss.

**Layout:**
- Hamburger button: visible on `md:hidden` (below 768px breakpoint). Positioned at nav right, replacing the language switcher's desktop placement.
- Overlay: `position: fixed`, `inset-0`, `z-index: 100` (above nav z-50), `bg-ink-950/95` (95% opacity ink-950 via Tailwind arbitrary: `bg-[rgba(13,13,26,0.95)]` or `bg-ink-950 bg-opacity-[0.95]`).
- Overlay animation: CSS transition `opacity 200ms ease-in-out`. Start at `opacity-0`, animate to `opacity-100` when open state is `true`. Use `visibility: hidden` when closed to remove from tab order.

**State:** `const [menuOpen, setMenuOpen] = useState(false)` in Nav component.

**Body scroll lock mechanism:**
- On open: `document.body.style.overflow = 'hidden'`
- On close: `document.body.style.overflow = ''`
- Cleanup in `useEffect` return function to restore overflow if component unmounts while open.

**Dismiss triggers:**
1. Close button tap (X icon, top-right of overlay, 44px min tap target)
2. ESC keydown: `useEffect` adds `keydown` listener when `menuOpen === true`, removes on close or unmount
3. Nav link tap: clicking any nav link closes the menu (call `setMenuOpen(false)` in link onClick)

**Close button:**
- Position: `absolute top-5 right-6` inside the overlay
- Visual: inline SVG X (16px), wrapped in a 44x44px button
- aria-label: "Close" (EN) / "Cerrar" (ES) — sourced from translations (add key `nav.menuClose` to translations.js)
- Class: `text-text-secondary hover:text-text-primary transition-colors`

**Overlay content layout:**
- Full-screen flex column, centered vertically and horizontally
- Nav links: stacked vertically, `text-2xl font-extrabold text-text-primary`, gap-8 between items
- Language switcher: same pill component as desktop, placed above nav links, `mb-10`
- No secondary content (no social links, no CV download in mobile menu)

**Hamburger button:**
- 3 horizontal bars, inline SVG, 24px icon
- aria-label: "Open menu" / "Abrir menú" — add key `nav.menuOpen` to translations.js
- aria-expanded: bound to `menuOpen` state
- aria-controls: `"mobile-menu"` — matches `id` on the overlay div

### Nav Scroll Spy + Active Indicator (NAV-03)

**Decision (user):** Color + bottom border — text-brand + 2px bottom border in brand color when scroll-spy detects active section.

**Mechanism:** `IntersectionObserver` watching each section's root element.

**Observer configuration:**
- `root: null` (viewport)
- `rootMargin: '-20% 0px -60% 0px'` — triggers when a section occupies the middle 20% of the viewport. This prevents the nav from flickering between states at section boundaries.
- `threshold: 0` — fire as soon as any part enters the rootMargin zone

**Sections observed:** `#about`, `#skills`, `#experience`, `#contact`. Each section element must have an `id` matching these anchors (section IDs are added in Phase 2 to each section component).

**State:** `const [activeSection, setActiveSection] = useState('')` in Nav component.

**Active link style:**
- Default: `text-text-secondary font-normal` (weight 400)
- Active: `text-brand font-normal border-b-2 border-brand pb-0.5` (weight 400, 2px border, 2px bottom padding to offset border from text baseline)
- Transition: `transition-colors duration-200` on the `<a>` element

**Edge case — page top (above #about):** When `activeSection === ''`, no nav link is highlighted. This is the correct state when the hero is in view.

### Scroll Progress Bar (NAV-04)

**Position:** `fixed top-0 left-0`, full width via `right-0`, `z-index: 200` (above nav z-50 and menu z-100).

**Dimensions:** height 3px. No border radius — flat bar spanning full viewport width.

**Color:** `background: var(--color-brand)` (#6C63FF). No gradient — solid brand color.

**Mechanism:** `useEffect` with `scroll` event listener on `window`. Calculates:
```js
const scrolled = window.scrollY / (document.documentElement.scrollHeight - window.innerHeight) * 100
```
Sets a `width` CSS property (0%–100%) on the bar element via a `useRef`.

**Performance:** Use `requestAnimationFrame` to throttle the scroll handler — update the ref directly (`progressRef.current.style.width = scrolled + '%'`) without triggering React re-render. No `useState` for the scroll position.

**Reduced motion:** The bar still updates width (it is not an animation, it is a direct state reflection). No suppression needed under `prefers-reduced-motion`.

**Implementation location:** `<ProgressBar />` sub-component inside Nav.js, rendered as the first child of the `<header>` element so it sits at the very top.

### Language Switcher (I18N-01, I18N-02, SEO-04)

**Desktop placement:** Nav right side, always visible at `md:flex` and above. Existing pill design retained: `bg-ink-500 border border-ink-400 rounded-full p-0.5`.

**Mobile placement:** Inside the full-screen overlay, above the nav links, `mb-10`. Same pill component — extract to a shared `<LangPill />` sub-component used in both desktop nav and mobile overlay.

**Persistence key:** `cam-lang` in localStorage — existing key, unchanged.

**Language flash fix (SEO-04):** Replace lazy `useState('')` initialization with synchronous read:
```js
const [lang, setLangState] = useState(
  () => (typeof window !== 'undefined' ? localStorage.getItem('cam-lang') : null) || 'en'
)
```
This eliminates the flash of English content before the stored language loads.

**Active state styling:** Active pill button: `bg-brand-gradient text-ink-900 font-extrabold`. Inactive: `text-text-secondary font-normal`. Transition: `transition-colors duration-150`.
