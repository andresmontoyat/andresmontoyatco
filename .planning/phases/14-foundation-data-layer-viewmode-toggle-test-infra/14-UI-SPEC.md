# Phase 14 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope of This Contract

Phase 14 is a **data + state + test foundation** phase. It ships **NO constellation rendering** — no
nodes, edges, experience cards, or filters. The design system (Tailwind 3 + CSS-var tokens, dark/light
themes, brand blue/emerald palette, Inter + JetBrains Mono, bilingual EN/ES) is **already established and
locked** in the live codebase; this contract does not re-design it.

**The only net-new UI surface in Phase 14 is the game/dev `ViewMode` toggle control** placed in the nav,
plus the minimal **game-mode placeholder shell** that the toggle reveals while the real render is built in
Phase 15. Everything else (constellation, clustering, cards, filters, WebGL/SVG renderers) is **deferred** —
see [Out of Scope / Deferred](#out-of-scope--deferred-to-phase-15).

This contract intentionally mirrors the existing `src/components/_shared/ThemeToggle.js` pattern: the
ViewMode toggle reuses the same tokens, spacing, focus styles, touch target, motion guard, and bilingual
`aria-label` conventions. Treat the established system as pre-validated; only the toggle + placeholder are
new design decisions here.

## Spacing Scale

The new toggle uses the project's existing Tailwind spacing (4px base). Declared values used in Phase 14
(all multiples of 4):

| Token | Value | Usage in Phase 14 |
|-------|-------|-------------------|
| xs | 4px | Icon/label gap inside the toggle (`gap-1`); pill inner padding (`p-0.5` mirrors LangPill) |
| sm | 8px | Spacing between nav controls — ViewMode toggle ↔ ThemeToggle ↔ LangPill (`gap-2`, existing nav cluster) |
| md | 16px | Default control padding; placeholder shell inner padding row (`px-4`-equivalent uses 16px) |
| lg | 24px | Placeholder shell vertical rhythm / section padding (`px-6` nav container is the existing 24px gutter) |
| xl | 32px | Placeholder shell content block gap |
| 2xl | 48px | Placeholder shell top offset below sticky nav |
| 3xl | 64px | Placeholder shell major vertical centering on tall viewports |

Exceptions:
- **44×44px minimum touch target** for the ViewMode toggle button — matches `ThemeToggle`'s
  `min-h-[44px] min-w-[44px]` and the existing mobile menu button (`w-11 h-11`). This is a deliberate WCAG
  2.5.5 / iOS HIG exception to the 4px scale and is REQUIRED for the icon control.

## Color

The 60/30/10 split is the locked, theme-aware system — values resolve from CSS vars and flip on
`[data-theme="light"]`. The ViewMode toggle and placeholder consume these tokens directly; they introduce
NO new color values.

| Role | Token (dark → light) | Usage |
|------|----------------------|-------|
| Dominant (60%) | `ink-900` `#12121F` → `#FAFAFA`; `ink-950` `#0D0D1A` → `#FFFFFF` | Nav backdrop (`bg-ink-900/70` + blur), game-placeholder full-bleed background |
| Secondary (30%) | `ink-500` `#2D2D5A` → `#C8C8D6`, `ink-400` border `#3A3A6A` → `#A8A8BC` | Toggle pill track + border (mirrors LangPill `bg-ink-500 border-ink-400`); placeholder card surface |
| Accent (10%) | `brand` `#3B82F6` → `#2563EB`; `brand-gradient` (blue→emerald) | Active toggle segment fill, focus ring, hover text |
| Text | `text-primary` / `text-secondary` / `text-muted` (theme-aware) | Inactive toggle label = `text-secondary`; active label on gradient = `text-ink-900` (inverse) |

**Accent reserved for (explicit list — Phase 14 only):**
1. The **active** segment of the ViewMode toggle (the currently-selected mode), using `bg-brand-gradient`
   + `text-ink-900` exactly as `LangPill` renders its active language.
2. The **focus-visible ring** on the toggle button (`focus-visible:ring-2 focus-visible:ring-brand`),
   identical to `ThemeToggle`.
3. The **hover** text color of the inactive control (`hover:text-brand`), identical to `ThemeToggle`.

Accent is NOT used for the placeholder background, body text, or any decorative fill. No destructive color
is needed — Phase 14 has no destructive actions.

| Role | Value | Usage |
|------|-------|-------|
| Destructive | not used | No destructive actions in Phase 14 |

**WCAG AA:** All token pairs above are already AA-verified in both themes (light `brand` `#2563EB` = 4.97:1
on white; light `accent` `#047857` = 5.05:1 on white; `text-muted` darkened to `#646478` for AA). The
toggle reuses these pairs, so AA holds in both modes by construction. The active-segment label uses
`text-ink-900` (`#0D0D1A`) on the blue→emerald gradient, matching the existing LangPill contrast treatment.

## Copywriting Contract

The toggle copy follows the design spec UX section ("Toggle (🎮 Juego | Dev)") and the existing nav i18n
pattern (`t.nav.*`). All copy is bilingual via `LanguageContext`; new keys live under the `nav` namespace
in `src/i18n/translations.js`, mirroring `themeLight`/`themeDark`.

| Element | Copy |
|---------|------|
| Primary CTA (the toggle itself) | EN: **Game** / **Dev** segment labels · ES: **Juego** / **Dev** — one-click switch between modes |
| Toggle `aria-label` (announces current → target) | EN: when in game → "Switch to developer view"; when in dev → "Switch to game view" · ES: "Cambiar a vista de desarrollador" / "Cambiar a vista de juego" |
| Toggle `aria-pressed` semantics | Each segment exposes `aria-pressed={isActiveMode}` (mirrors LangPill EN/ES `aria-pressed`) so SR announces which mode is selected |
| Game-mode placeholder heading | EN: "Game mode is loading" · ES: "El modo juego está cargando" |
| Game-mode placeholder body (+ next step) | EN: "The interactive skill constellation arrives next. Switch to Dev view for the full portfolio." · ES: "La constelación interactiva de habilidades llega pronto. Cambia a la vista Dev para ver el portafolio completo." |
| Empty / no-data state | EN: "No skill data available." (defensive only — graph derivation owns real data; shown only if the derived graph is empty) · ES: "No hay datos de habilidades disponibles." |
| Error state | EN: "Couldn't load game mode. Switch to Dev view to see the portfolio." · ES: "No se pudo cargar el modo juego. Cambia a la vista Dev para ver el portafolio." (graceful fallback — never a broken screen; mirrors spec Error Handling) |
| Destructive confirmation | none — Phase 14 has no destructive actions |

**sr-only / SEO obligation (contract note, full delivery is Phase 15 / GAME-06):** Because game mode is the
**default landing**, the game subtree MUST keep a crawlable, screen-reader-readable text fallback in the DOM
even at the placeholder stage — at minimum the placeholder heading + body above as real (not `aria-hidden`)
text, so crawlers/ATS never see an empty default landing. The full visually-hidden semantic experience list
is Phase 15's obligation; Phase 14 only must not regress crawlability of the default route.

## Interaction & A11y Contract (ViewMode toggle)

This is the load-bearing new interaction contract for Phase 14. It mirrors `ThemeToggle` and `LangPill`.

| Dimension | Contract |
|-----------|----------|
| Placement | In the nav control cluster, next to ThemeToggle + LangPill (`<div className="hidden md:flex items-center gap-2">`); also mirrored into the `MobileMenu` control row, exactly as ThemeToggle/LangPill are duplicated there |
| Control shape | Two-segment pill (Game · Dev) following `LangPill` (rounded-full track, `bg-ink-500 border-ink-400`, active segment `bg-brand-gradient text-ink-900 font-extrabold`), OR a single icon button following `ThemeToggle`. **Pill (segmented) is the recommended default** because there are exactly two labeled modes, matching LangPill's EN/ES affordance. Executor may collapse to a compact pill on mobile per spec ("Toggle compact at top"). |
| Touch target | ≥ 44×44px (each interactive segment / the button) — required |
| Keyboard | Fully keyboard-operable: `Tab` to focus, `Enter`/`Space` to activate; segments are real `<button type="button">` like LangPill (no custom key handling needed) |
| Focus | Visible `focus-visible:ring-2 focus-visible:ring-brand focus:outline-none` (token-matched to ThemeToggle) — must be visible in both themes |
| Accessible name | `aria-label` announces the **target** action (current mode is conveyed via `aria-pressed`), bilingual; see Copywriting table |
| Motion | Any color/segment transition uses `motion-safe:transition-colors duration-200` (mirrors ThemeToggle); under `prefers-reduced-motion: reduce` the global `index.css` rule already drops transitions to ~0.01ms — no extra work, but the toggle MUST NOT introduce a non-`motion-safe` transition |
| Theme parity | Renders correctly in dark and light via tokens only; no hard-coded hex |
| Persistence (state, not visual) | `localStorage` key `cam-viewmode` (follows `cam-*`), `?mode=` deep-link, default = game, `typeof window` guard — owned by `ViewModeContext`, mirrors `ThemeContext` |
